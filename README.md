# Amazon E‑Commerce Sales — Pandas Cleaning & PostgreSQL Load

This project demonstrates a complete **data engineering workflow** performed on the Kaggle Amazon Sales Report dataset. The goal of this project was to **clean the raw data**, **split it into three logically structured datasets**, and **store the final processed tables inside a PostgreSQL database** for further analytics and BI integration.


End‑to‑end data engineering workflow using **Pandas** + **SQLAlchemy** to clean the Kaggle [Amazon Sales Report dataset](https://www.kaggle.com/datasets/thedevastator/unlock-profits-with-e-commerce-sales-data?utm_source=chatgpt.com) , segment orders into **revenue**, **leakage**, and **in‑transit** groups, and write them to **PostgreSQL** tables for downstream analytics.

---

## 📦 Dataset

- **Source**: Kaggle
- **Raw file used**: `Amazon Sale Report.csv`  
- **Raw shape**: `128,975 rows × 24 columns`
